Following are the insights gathered from this initial analysis:

  • The entire dataset contained 500,000 instances, and following a 60%-40% train-test split, 300,000 instances were assigned to the training set and 200,000 instances for the test set. The test dataset consisted of 31 columns, excluding the target. The 32 variables included in our training dataset were of the following types: ID, categorical (from 0 to 18), continuous (from 0 to 10) and target.
  • As the observations we were to work with were insurance records, the ID column assigned a number to each client in order to comply with the anonymity requirements, but it was excluded from the final training and testing sets. Columns with the label ‘cat’ contained multiple types of values, which were in the form of letters (i.e. A, B or AA), and represented the categorical attributes relative to each customer.
  • The columns with numerical data were labelled as cont1, cont2 until cont10, with values between 0 and 1, that quantifies in probabilistic terms some unknown risks, clearly defining each subject’s profile as well as his/her risk aversion. As mentioned above, only the training set had an inexplicit target column with binary values (0’s and 1’s), which could be used to predict two different insurance policies to be proposed or a final verdict such as, risky-not risky, on sampled individuals.

Our next step is to check the different levels for each factor covariate. The reason behind this action is that in Python the testing and training set must have the exact number of factor levels for the identical covariate. When we apply 1-hot-encoder, each level will act as a unique covariate. With this being said if one factor covariate has different number of levels between the two sets, the design structure will not be correct.

1-Hot-Encoder is a method used in Python to overcome the different levels of the categorical (string) columns of the data. Given that Python models only understand numerical values, we need to convert the string data to numerical without implementing a mathematical meaning to them. This technique creates a new column for each different level and fills the cells with binary values (1 = true, 0 = false).

Random forest is another one of the three models that were handpicked to be used for this dataset. This model is a commonly used classification algorithm that can be used for classification and regression tasks. It also works well with large data sets similar to the one we are using, which makes it a suitable choice. As the name suggests, the random forest algorithm is a collection of decision trees. It has access to all the hyperparameters available for a decision tree model and bagging classifier, giving more control over the model creation. Similar to the other models chosen, the random forest is comparatively robust to outliers and works relatively well with linear as well as non-linear data, owing to its random nature. Given that the model merges multiple decision trees and works to produce the average of the result obtained, it generates stable and accurate results through cross validation. In general, using more trees results in a more robust prediction that has a higher accuracy.

  • Logistic Regression

Logistic regression can be used for supervised classification problems since it relies on a linear regression equation to generate binary outputs. It is absolutely efficient to train and its results are easier to interpret than the ones from more complex algorithms such as Neural Networks. Moreover, as a classifier, it is less prone to overfitting, which means overstating the accuracy of predictions on the training set so that the model does not predict accurately on the test set. Within its framework multicollinearity is not an issue since it can be tackled by opting for L2 regularization. Although it is convenient while dealing with probabilities for observations, it does not perform well when covariates’ space is large, particularly if categorical features are numerous. Another core limitation of logistic regression is the assumption of linearity between the label (target) and the independent variables, despite accounting for non-linear features via transformation. However, we wanted to test if this model would perform well in comparison with the others, thus testing its limitations.

  • XGBClassifier

XGB (short for eXtreme Gradient Boosting) is one of the newest and most presiding classifier models currently available. This model is based on gradient boosted decision trees, combining C++ code and the python interface for a robust yet easy to use model. A gradient boosting model is where models are trained successively, using the previous model's error to adjust the next model. The XGB Classifier model is optimized for computational speed and to get a high performing model and has been dominating online competitions such as Kaggle. It is also able to work effectively on large datasets and data sets that feature outliers. One of the only problems with XGBoost is that with all the parameters it can be difficult to perfect the fine-tuning for optimal results, which could lead to overfitted results. Irrespective of this limitation, we chose this model to predict our target response, to ensure that we utilized the foremost technology available to us in this field.

[3.2] Reducing dimensions by exploring feature importance¶

From the data wrangling and convertions applied above, the final dataset used for fitting and making preditions is a large sparse matrix that consists of 300K rows and 642 columns. This can cause issues in terms of time and computational complexity. For this reason, we have chosen to apply a thorough analysis on feature important analysis and continue the model fitting with the results from this analysis i.e., with only the features that contribute to our response.

Our goal for this assessment of the given dataset is to reduce the dimensions and keep a similar (if not the same) percentage of accuracy for all our models. In the case of having multiple different models that have the same predictive power, it is beneficial to proceed with the less complex ones. Reducing dimensions of the training data also reduces the complexity of the model.

By decreasing the dimensions of the dataset we achieve a lower accuracy score. However, dimensionality reduction consists of a sharp decrease in computational complexity and allows time saving. ### FUNCTION that one hot encodes the 2 datasets together
## Merged dataset will be send and it will be returned 1-Hot-Encoded
def find_one_encoder(train, test):
    ## MERGED DATA
    # This will concatenate the test data rows below the train data
    # The first 300K rows will be the train
    # The last 200K rows will be the testing
    merged_encoder = pd.concat([train, test])


    # Using get_dummies() which is the same thing as 1-hot-encoder but ignores numerical values
    merged_encoder = pd.get_dummies(merged_encoder)
    return(merged_encoder)
